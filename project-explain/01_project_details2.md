# ✅ **Simple, Non-Technical Project Explanation**

### **Project Name: Employee Compliance Management System**

### **1. What the project does (Simple Explanation):**

This system helps a company keep track of all its employees’ documents—like ID proof, certificates, bank details, etc.
It also keeps checking whether any document is missing, expired, or needs an update.

Think of it like a digital folder for every employee that always stays up-to-date.

---

# **2. Who uses this system?**

* **HR Team**
* **Managers**
* **Auditors**

They use it to quickly see:

* Which employee has all required documents
* Which employee still needs to submit something
* Which documents are about to expire

---

# **3. What problems does this project solve?**

Before this system, HR had to check everything manually in Excel sheets or emails.
That took a lot of time and led to mistakes.

With this system:

* Everything is **automated**
* HR gets updates instantly
* Company stays audit-ready

---

# **4. How the system works (in simple terms):**

### **Step 1: Employee Uploads Document**

Employees upload their documents in the portal.

### **Step 2: System Stores and Organizes**

The system organizes the documents and links them to the correct employee.

### **Step 3: System Checks for Issues**

The system automatically checks:

* Is any document missing?
* Is any document expired?
* Is any document about to expire?

### **Step 4: Notifications**

HR and employees get reminders through email/portal.

### **Step 5: HR Dashboard**

HR sees a clear dashboard:

* How many employees are compliant (complete)
* How many are non-compliant (missing items)

---

# **5. Technologies (Explained in easy words):**

### **Backend (Java + Spring Boot)**

This is the brain of the system.
It handles:

* Saving data
* Checking compliance
* Sending notifications

### **Frontend (If applicable)**

This is the screen HR and employees see.

### **Database**

This is the storage place where all employee data and document info is saved.

### **Kafka (in simple words):**

Kafka is like a courier service inside the system that delivers messages between different parts of the application.
For example:

* When a document is uploaded → Kafka sends a message to trigger compliance check.

### **Liquibase (in simple words):**

Liquibase is used to manage database changes safely.
If we need a new table or a new column, Liquibase makes sure the change is applied in all environments without breaking anything.
